Cluster 1 — From Anthropic's Own Multi-Agent Research System
Opus lead + Sonnet subagents
This multi-model pattern outperforms single-agent Opus by 90.2% on breadth-first queries. Validates our multi-model choice.
3–5 parallel subagents
Teams of 20+ underperform. Planner should generate 3–5 sub-questions max per level. We should enforce this as a config param.
Token efficiency = 80% of variance
Keep BranchGraph context minimal. Don't pass full research tree to leaf nodes — only the branch question + parent summary.
LLM-as-judge dimensions

Cluster 2 — LangGraph-Specific Best Practices
Send API for map-reduce
Native pattern for fan-out. Return a list of Send(node, state) objects from fan_out node. LangGraph scheduler runs them concurrently.
Keep sub-agent state small
Node-specific states should contain only fields the node produces. Don't bloat BranchState with full parent context.
Built-in checkpointing
LangGraph's MemorySaver gives us free checkpoint-based error recovery. Resume from last good node on failure instead of restarting entirely.
Observability from day one
LangSmith tracing (free tier) for production debugging. Structured logs per node event. Non-determinism is hard to debug without traces.
Cluster 3 — Design Changes This Triggers
⚠
Add max_branches config (was implicit, now explicit)
Planner must be instructed to generate exactly N sub-questions (default 3–5). Add max_branches: int = 4 and max_depth: int = 2 as ResearchConfig fields.
⚠
BranchGraph gets only branch question + parent summary (not full state)
Original design passed too much context. Each BranchGraph input: {question, depth, parent_summary, config}. Keeps token count down, improves reliability.
✚
Add LangGraph MemorySaver checkpointing
Wire in LangGraph's built-in checkpointer. On /research/:id/result endpoint, check checkpoint state before running — enables resume-on-failure for long research jobs.
✚
Richer eval dimensions in scripts/eval.py
LLM-as-judge scores 5 dimensions per answer: factual accuracy, citation accuracy, completeness, source quality, tool efficiency. Report per-dimension breakdown, not just pass/fail.
✚
LangSmith tracing (optional, free tier)
Add LANGSMITH_API_KEY env var support. If set, all graph runs are traced automatically. Strong demo talking point: "here's a full trace of that research run."
Bottom line: Our architecture is well-aligned with best practices. The main refinements are: enforce max_branches/max_depth explicitly, slim down BranchGraph input state, add MemorySaver checkpointing, and enrich the eval with multi-dimension LLM-as-judge scoring. The Anthropic article is a direct endorsement of the multi-model orchestrator+worker pattern we already chose.
