A — Collapsible Tree (Sidebar)

Left panel shows a collapsible indented tree. Click a node to open its detail panel on the right — sources, sub-questions, excerpts. Clean, familiar, easy to scan. Like VS Code's file explorer meets Perplexity sources.

Pros

  • Very scannable
  • Low implementation complexity
  • Works well on any screen size

Cons

  • Less visually striking
  • Doesn't convey parallel branching well

B — Live Force Graph (Center Stage)

An animated node graph (React Flow or D3) fills the main canvas. Nodes appear in real time as agents spawn branches. Click any node to expand a drawer showing that branch's sources and reasoning. Pulsing ring = currently being researched.

Pros

  • Most visually impressive
  • Shows parallel work beautifully
  • WOW factor for the demo

Cons

  • More complex to implement
  • Can get cluttered for deep trees

C — Split Panel (Tree + Detail)

Left panel = collapsible tree. Right panel = selected node detail (sub-questions it generated, sources found, excerpt previews, and a "Dive deeper" button to spawn a new agent on that node). Best balance of information density and clarity.

Pros

  • High information density
  • Clear drill-down UX
  • Closest to Perplexity + more

Cons

  • Less visually dramatic than B

My recommendation: C for the initial submission (high signal-to-noise, clear UX), with B's live animation as an optional view toggle if you have time.
