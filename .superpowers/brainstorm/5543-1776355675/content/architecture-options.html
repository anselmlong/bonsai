Overall System Architecture — 3 Approaches

How the LangGraph agent graph, FastAPI, and Next.js frontend connect

Sequential depth-first tree
① Planner Node → sub-questions [ Q1, Q2, Q3 ]
↓ one at a time
② Researcher(Q1) → sources + maybe sub-Q
↓
③ Researcher(Q1a) → deeper sources
↓ then Q2, Q3...
④ Synthesizer → final answer

A — Sequential Depth-First

Planner emits sub-questions; researcher handles them one at a time, recursing before moving to the next branch. Simple state machine, easy to reason about.

Pros

  • Easiest to implement and test
  • Predictable streaming order
  • Low token overhead

Cons

  • Slow — no parallelism
  • Boring to watch in the UI
  • Doesn't showcase LangGraph well
Parallel fan-out via LangGraph Send API
① Planner → Send(researcher, Q1) Send(researcher, Q2) Send(researcher, Q3)
R(Q1)
⟳ live
R(Q2)
⟳ live
R(Q3)
⟳ live
↓ each can Send() sub-branches
② Reducer collects all → Synthesizer

B — Parallel Fan-Out (LangGraph Send)

Uses LangGraph's native Send API to fan out to parallel researcher nodes. Each researcher can recursively Send sub-branches. A reducer waits for all branches before synthesis.

Pros

  • Fast — true parallelism
  • Live branching looks great in UI
  • Native LangGraph pattern

Cons

  • Harder to control max depth
  • Reducer complexity
  • More tokens in flight simultaneously
Hierarchical subgraphs — RECOMMENDED
ResearchGraph (root)
planner → fan_out → reduce → synthesize
BranchGraph (subgraph, max_depth)
search → reflect → recurse?
BranchGraph (depth+1)
SSE stream: node_started, node_result, node_complete events

C — Hierarchical Subgraphs ⭐

Root graph handles planner + synthesis. Each branch is a reusable BranchGraph subgraph that can recursively spawn child BranchGraphs up to max_depth. Every graph node emits typed SSE events the frontend consumes to build the live tree.

Pros

  • Clean separation of concerns
  • Reusable branch unit = easy to test
  • Depth control is explicit
  • Best interview architecture story
  • SSE events map 1:1 to UI nodes

Cons

  • More upfront design work

All 3 use FastAPI SSE → Next.js EventSource. The difference is purely in how the LangGraph graph is structured internally.
