# Bonsai — Design Context

## Design Context

### Users
Researchers, analysts, and technical professionals running complex multi-topic queries. Used in focused, often solitary sessions — late at night, in dim environments, during deep work. The job to be done: understand a complex topic thoroughly, with full visibility into *how* the research was conducted and *which sources* informed each conclusion. Users want to trust the output, which means they need to see the process.

### Brand Personality
**Precise, deep, calm.** A tool for serious thinkers. Quiet confidence, no noise. Like a well-made scientific instrument — every element is there because it needs to be. References: Linear (dense, keyboard-first, no decoration) + Perplexity (source transparency, research feel). Anti-reference: generic AI chatbot UIs with glow effects, gradient text, glassmorphism.

### Aesthetic Direction
- **Theme**: Light. Cream/paper backgrounds with warm undertones.
  - Background: `#f5f0e8` (warm cream)
  - Surface: `#ece6d8`, `#e0d8c8`
  - Border: `#d4c8b4`, `#c0b8a8`
- **Accent**: Forest green (`#3d6b4a`) — scholarly, botanical, calm. Matches the "Bonsai" brand.
  - Red (for errors): `#8b3a2a`
- **Typography**:
  - UI / headings / tree nodes: **Bricolage Grotesque** (Google Fonts, variable) — geometric grotesque with subtle optical character. Machined, not friendly.
  - Body / summaries / excerpts: **Spectral** (Google Fonts, Production Type) — screen-optimised serif, scholarly-editorial quality. Italic for source excerpts.
  - Monospace / tree labels / scores / status: **Azeret Mono** (Google Fonts) — precise without being generic IBM Plex.
- **Layout**: High information density. Left-aligned. Asymmetric. The tree panel and detail panel feel like a professional IDE, not a consumer app.
- **Motion**: Minimal. Node appearance in tree = fade + translate-y. Active/searching nodes = subtle opacity pulse (not a spinning loader). No bounces, no elastic easing.

### Design Principles
1. **Earn every pixel.** If it doesn't carry information or aid navigation, remove it.
2. **Trust through transparency.** The user should always know what the agents are doing — surfaced via status labels, source counts, and depth indicators, not hidden behind loading spinners.
3. **Depth without overwhelm.** The tree can be complex; the UI reveals complexity progressively (collapsed by default, expanded on interaction).
4. **Precise over pretty.** Functionality over decoration. The aesthetic emerges from tight spacing, good type, and restrained color — not from effects.
5. **Keyboard-friendly.** Power users should be able to navigate the tree, select nodes, and trigger actions without touching the mouse.
