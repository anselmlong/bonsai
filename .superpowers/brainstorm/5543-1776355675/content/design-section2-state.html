Design — Section 2: State Schema & Node Events

The data structures that flow through the graph and stream to the frontend

ResearchConfig — top-level settings, read-only throughout graph
class ResearchConfig(TypedDict):
    max_branches: int          # hard cap on sub-questions per level (default 5)
    max_depth:    int          # hard cap on recursion depth (default 2)
    planner_model:  str        # e.g. "gpt-4o"
    researcher_model: str      # e.g. "gpt-4o-mini"
    synthesizer_model: str     # e.g. "gpt-4o"
    tavily_max_results: int    # results per Tavily call (default 5)
ResearchState — root graph state
class ResearchState(TypedDict):
    job_id:        str
    query:         str
    config:        ResearchConfig
    sub_questions: list[str]          # emitted by planner
    branches:      list[BranchResult]  # collected by reduce node
    final_answer:  str                 # emitted by synthesizer
    events:        Annotated[list[NodeEvent], operator.add]  # append-only
BranchState — subgraph state (kept small — only what branch needs)
class BranchState(TypedDict):
    node_id:        str               # unique ID for this branch node
    parent_id:      str | None        # None for root branches
    question:       str               # the sub-question this branch answers
    parent_summary: str | None        # context from parent (not full state)
    depth:          int
    config:         ResearchConfig
    search_results: list[SearchResult]
    sub_branches:   list[BranchResult]  # collected if recursed
    summary:        str               # branch's synthesized answer
    sources:        list[Source]
NodeEvent — every SSE message the frontend receives
class NodeEvent(TypedDict):
    type:      Literal[
                 "research_started",    # job kicked off
                 "plan_complete",       # planner emitted sub_questions
                 "branch_started",      # BranchGraph began for a question
                 "branch_searching",    # Tavily call in progress
                 "branch_reflecting",   # reflect node deciding whether to recurse
                 "branch_spawning",     # recursing — child branches created
                 "branch_complete",     # branch finished, summary + sources ready
                 "synthesis_started",   # all branches done, synthesizer running
                 "research_complete",   # final answer ready
                 "error"
               ]
    node_id:   str
    parent_id: str | None
    depth:     int
    question:  str | None
    sources:   list[Source] | None    # present on branch_complete
    summary:   str | None            # present on branch_complete
    answer:    str | None            # present on research_complete
    timestamp: float
Source & BranchResult — leaf data types
class Source(TypedDict):
    url:     str
    title:   str
    excerpt: str
    score:   float  # Tavily relevance score

class BranchResult(TypedDict):
    node_id:  str
    question: str
    summary:  str
    sources:  list[Source]
    depth:    int
How the frontend reconstructs the tree from events alone
// On each SSE message:
branch_started → add node to tree (parent_id tells us where)
branch_searching → set node status = "searching" (pulse animation)
branch_reflecting→ set node status = "reflecting"
branch_spawning → show child nodes will appear
branch_complete → set node status = "done", attach sources + summary
research_complete→ render final answer panel
// No other state needed. node_id + parent_id = full tree structure.
