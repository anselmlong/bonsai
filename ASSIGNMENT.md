Build a multi-agent system that does deep research given a user query.

- You may use any libraries and frameworks.
- You are free to use any AI tools. However, keep in mind that during the technical interview in the next stage, you’ll be expected to answer questions about the codebase.
- This is deliberately open-ended and is up to you to decide what this should look like, and be. Place it behind a fastapi backend server and call it via a standard HTTP request, or run it via a CLI, or have a UI with SSE; the choice is yours the way the user can make a query.
- No frontend is required; but you should allow a way for the user to submit a query (e.g. via curl, or a clear README to run your script).

Submission:

- Provide a Github link to your project. Your repository should minimally have a README. Any demos are welcomed: e.g. a live link, video, etc.

You will be assessed on the following criteria:

- the correctness of its output (stochastic functions notwithstanding);
- how testable, modular and clean your code is;
- other interesting add-ons you can think of.
